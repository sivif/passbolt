🔍 Introduction to Password Managers
  • What is a Traditional Password Manager? A digital vault solution designed primarily for individual or centralized credential storage.
  • Core Functionality: Acts as a locked repository for storing web logins, secure notes, and sensitive credentials.
  • Standard Workflow: Users authenticate with a single Master Password to unlock a personal database usually managed by a third-party cloud.

🔍 Introduction to Passbolt / Use cases
  • What is Passbolt? An open-source, team-first, zero-knowledge password manager designed for technical and enterprise environments.
  • The Two-Gate Security Model: Utilizes local asymmetric OpenPGP keys. Your Master Password unlocks your unique local Private Key right inside your browser extension to unscramble your passwords. The actual server never sees your master password or unencrypted secrets.
  • Key Use Cases:
    • Data Sovereignty: Organizations looking to self-host their own vault so corporate logins never touch a third-party server.
    • Granular Vault Management: Sharing access to a single password with specific team members without giving away control of an entire folder.
    • Securing Infrastructure: Storing individual API keys, server SSH credentials, or config profiles away from shared chats.

🔍 Passbolt FAQs — Trust & Everyday Use
⚠ Why is there no "Forgot Password" or "Reset via Email" link?
Your Master Password functions as a real-world physical key. Because Passbolt does not store your key on their servers, they have no physical way to reset it for you. If you forget it, the vault remains locked permanently to preserve total privacy.
Q: Can IT administrators or Passbolt view my passwords?
A: No. Passwords are locked and scrambled right on your own machine *before* they travel over the internet. Admins only see unreadable, scrambled gibberish blocks.
Q: Why force a browser extension instead of a basic website?
A: Websites can be easily copied or tampered with by hackers. A browser extension lives locally inside your computer's browser, providing a much stronger protective shield against phishing tricks.
Q: Can a hacker steal my entries by monitoring my Wi-Fi data?
A: No. Since your vault is completely encrypted before it ever leaves your device, any attacker eavesdropping on the network connection will only capture locked, useless data packages.
Q: Can I use it on my smartphone? How do keys transfer?
A: Yes. You link your phone app by scanning a secure QR code from your computer screen, which copies your private security key safely without passing it over the open internet.

🔍 Passbolt FAQs — Outages & Network Realities
⚠ Critical Fact: What happens if the server crashes or Wi-Fi blocks it?
Passbolt behaves on a strict server-client system and does not save passwords offline. If the network blocks the connection or the server turns off, the browser extension will freeze on a "Connecting..." loop or network error and completely lock you out until link access is restored.
Q: What if the self-hosted storage server gets hacked?
A: The intruders gain absolutely nothing. Because they lack your personal Master Password and Private Key file, the stolen database remains an unbreakable wall of code.
Q: Why is my extension stuck infinitely spinning on "Connecting..."?
A: Your local network or router is dropping communication to your server. The extension keeps waiting out in the open for a handshake reply that never reaches it.
Q: If I clear my browser cookies or history, does it delete my passwords?
A: No. Passwords live safely inside the main system database. Clearing browser history just unlinks your local extension session, requiring you to perform a quick profile import to get back in.
Q: What if our server's IP address changes?
A: The extension will lose track of where the server lives and drop connectivity. An admin must configure a stable web name (domain) instead of numbers to avoid this.

🔍 Passbolt FAQs — Team & Administration
💡 Why use Passbolt when free browser password managers exist?
Browser managers (Chrome, Safari) are built for single users and pass all passwords to a corporate cloud. Passbolt is built specifically for **collaborative teams**, allowing you to share keys with specific coworkers without exposing master records, all while keeping 100% data control inside your server.
Q: What is the "Recovery Kit" (.asc) file for?
A: Since the server does not store your master key, this file is your personal backup piece. If you want to log into your vault from a brand new computer, you must upload this file to open entry.
Q: Can a user log in with a password without viewing it?
A: Yes. Admins can grant "Use Only" rights. The browser extension will automatically fill the password into the website fields for the analyst without ever displaying the raw password characters.
Q: How do we onboard new SOC analysts safely?
A: Admins send an invite code, users generate their own security key pairs locally, and an administrator manually approves their signature match to establish verifiable trust.
Q: Does Passbolt track who touches or views a password?
A: Yes. It runs a permanent, unalterable log tracking precisely which user updated, checked, or passed a credential string along, fulfilling strict security compliance laws.
