🔍 FAQs of PassMan
Q: Where is data stored?
A: PassMan typically relies on centralized cloud or local database storage depending on the deployment model.
Q: How are backups handled?
A: Backups are usually handled at the database level by the system administrator.
Q: Does it support granular sharing?
A: Sharing is often limited to folder-level access or basic direct sharing without deep cryptographic verification per user.

🔍 Introduction to Passbolt / Use cases
  • What is Passbolt? An open-source, team-first password manager designed specifically for Agile and DevOps teams.
  • Security Model: Utilizes end-to-end encryption using OpenPGP. The server never sees the unencrypted passwords.
  • Key Use Cases:
    • SOC Teams: Securely sharing critical incident response credentials.
    • DevOps: Integrating secrets into CI/CD pipelines via API.
    • IT Departments: Role-based access control (RBAC) for infrastructure passwords.

🔍 What Ifs / FAQ to Passbolt
⚠ What if I lose my private key?
Passbolt relies on OpenPGP. If you lose your private key and haven't configured an account recovery protocol, you cannot recover your account. Always maintain secure backups of your key.
Q: What if the self-hosted server goes down?
A: The browser extension caches credentials locally for offline access (if enabled), allowing temporary read-only access until the server is restored.
Q: How do we onboard new SOC analysts securely?
A: Admins invite users, users generate their own PGP keys locally, and admins cryptographically verify their identity before granting vault access.
